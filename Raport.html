Reporting Tool

Author

  • Bitiusca Madalina

Reporting Tool is an application designed to identify problems in a specific area. The reported problems will be classified and verified in order to be solved.

1. Introduction

A web application is implemented for reporting environmental issues within a tourist / urban area. The most important aspects concern pollution, the existence of garbage and / or waste, anti-civic behavior, any other events / accidents with an impact on the environment and others.

2. Actors and goals

2.1 User

Can create an account, manage it (change password or add a profile picture), report a pollution issue after logging in (no login, no access to reporting), make suggestions using the form on the Contact page

2.2 Admin

Keeps the information on the site up to date, manages the site, maintains the database, checks problems reported by users.

2.3 Interface

Gives the user an easy way to view the site

2.4 Database

Data storage (stores the proposed issues, user accounts and possibly their information), retains the information in the most efficient way possible for fast searches, is maintained by the administrator

3. Operating environment

The application is designed to be used on anything device that has a web browser.

4. Use cases

4.1 The user wants to view the site / suggest a problem

4.1.1 Objective / Context

The user needs to know the address of the page, can view it but needs to log in to report a problem.

4.1.2 Scenario / Authentication steps

The user will have access to a login form, if he does not already have an account, he can create it with the help of the e-mail address (he will receive a confirmation code on the e-mail address to activate the account)

After creating your account, enter your username and password in the corresponding fields, having, in case of successful login, access to the problem reporting form

Any user will be able to customize their profile, add information and / or a profile picture

Users will be able to contact the site administrator to bring suggestions for improvement / opinions through the form on the "Contact" page

4.1.3 Extensions

If the user forgets their username and / or password, has the possibility to be "reminded" through the e-mail address.

4.2 The user fills out the problem reporting form

4.2.1 Objective / Context

The user is logged in and provided with a form to report a problem.

4.2.2 Scenario / steps

The user logs in following the steps from 4.1

The user fills in the fields with his name, surname, type of problem and his description, having the possibility to upload and representative images, then press "Send".

The user is notified if the form has been successfully sent via a message.

4.2.3 Extensions

If the user skips one (or more) of the fields, a warning message will appear stating that all fields are required.

If the user wants to leave the site before giving "Send", a warning message will be displayed.

4.3 Receiving and processing information

4.3.1 Objective / Context

Users have completed the form and the data is stored in the database.

4.3.2 Scenario / steps

The administrator is logging in

The administrator will have access to some specific tables, will be able to view the list of users and problems proposed by each

The administrator can delete a user's account
